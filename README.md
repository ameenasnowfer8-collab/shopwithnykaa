# 💄 Nykaa E-Commerce Database Management System

## 📌 Project Overview

This repository contains the development of the **Nykaa E-Commerce Database Management System**. The project focuses on designing a relational database for an online beauty and personal care platform. It covers requirement analysis, database planning, entity analysis, relationship design, and ER diagram creation to build a strong foundation for an efficient e-commerce database.

## 📅 Project Progress

### 🗓️ Week 1 – Project Planning & Requirement Analysis

* Selected the project topic.
* Defined the project objectives and scope.
* Prepared the problem statement and business requirements.
* Identified stakeholders.
* Documented functional and non-functional requirements.

### 🗓️ Week 2 – Database Design

* Identified the database entities.
* Defined attributes for each entity.
* Assigned primary keys and foreign keys.
* Selected appropriate data types.
* Applied database normalization concepts.

### 🗓️ Week 3 – Entity Analysis & ER Diagram

* Finalized entity relationships.
* Defined one-to-one and one-to-many relationships.
* Created the Entity-Relationship (ER) Diagram.
* Designed the database schema using **dbdiagram.io** and **Draw.io**.
* Verified data integrity through primary and foreign key relationships.

## 🗂️ Main Entities

* Customer
* Product
* Category
* Brand
* Cart
* Orders
* Order Details
* Payment
* Review
* Admin

## 🔗 Key Relationships

* Customer places Orders
* Customer adds Products to Cart
* Category contains Products
* Brand manufactures Products
* Orders contain Order Details
* Orders have Payments
* Customer writes Reviews
* Product receives Reviews
* Admin manages Products
* Admin manages Orders
